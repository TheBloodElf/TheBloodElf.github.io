公司项目流程

这篇文章是写给找工作的人的，因为面试时候会问到这个问题，那么公司的项目流程是什么样的呢？
一：先有点子，点子是谁提供的呢？这个得看公司的制度了，是专人或者所有人集合一起。
二：有了点子不是就可以开始动手了，得先大概的想一想这个方案能不能做出来？这时候需要技术人员和需求人员等相关人一起讨论。
三：如果方案不可行，则另外想或者减少方案中复杂的部分。
四：如果方案可行，那么就可以动手了（除了客户端），比如原型设计师做原型，我们公司用的原型设计工具是Briefscase；服务器的人就可以做接口了（需要和客户端的沟通）；如果项目是有其他部分的，比如微信公众号、网页端等等，那么这些人也可以开始动手了，因为这些往往是客户端的一个链接进去或者跳转过去的。
五：那么客户端的人做什么呢？他们开始封装一些要用的工具类（二维码等等）和集成一些功能（地图、第三支付等等），然后等到原型出来就可以开始做界面了，然后等服务器做一部分接口就开始对接接口填充数据了。
六：做的时候客户端人员需要喝原型设计师沟通，就是给它们开开是不是这个样子？不是就改！如果公司测试模式是“双v”等，则需要边测试边写或者什么的。
七：项目出来了，这时候需要一次集中测试，看有问题没有，没有的话就可以上线了！
八：上线了，不是就没事情做了，这时候会得到客户的一些反馈（如果我不喜欢自己的照片给别人看），那么相关功能的人员就需要更新功能了（这就是迭代吧）。
九：经过一定量的八步骤，可能就需要更新版本了，就是改了再提交审核。

附加：
一：如果你是公司才招进去的，那么你开始的任务是熟悉上一个版本的框架和软件大概（不是让你看懂所有的代码），然后可能需要重新从头写所有的程序（所以没必要把他的思想看很彻底，反正都要自己写），如果不是从头写，那么你就需要好好的看别人的代码了（很痛苦！！！）。
二：你一般是有一个直接上级的，那么你的工作要定期给他讲，这样才能让他了解你，这样才能合理利用资源嘛！
三：如果你是才进入的，那么核心的代码不会让你写的（除非只有你一个程序员），上级会安排一些简单的任务给你，你得高质量完成！等上级给你权利写核心代码的时候，你就已经很厉害了！那么有人就问了，我写简单的对自己没进步啊。那么你就是真的没有工作经验了（工作的人都知道，除了自己特别懒，总感觉时间不够，ps：我都有好多东西想学呢）。
四：代码同步等必备工具，好吧这些你得花两个小时好好看看（只需要两个小时），不要在这上面体现的自己多菜。
五：上级安排你的任务，你不会就谷歌（一定是谷歌），不要总问别人，别人其实都会，你这样去公司是学技术去了？（没人希望新员工是这样，除非BAT等，人家有钱，就爱”培养“）
六：。。。（好吧写不完）

好了，这些就是我的初步理解了（因为我才工作一个月，有错误我后面慢慢改）

自己做微信图片浏览demo

实现方式是非常多的，其实昨天我就开始做这个东西了，但是发现网上的都太复杂了，而且自己也用不了那么多的功能，于是我就自己动手做一个吧！先看一下文件列表：


Photo文件是图片对象，可以放图片的地址或者数据；
AllViewController是点击右下角出现所有图片显示的视图；
BorewViewController是左右滚动的视图；
BorewScrollView是每个在BorewViewController中滚动的item对象。
那么我们这样使用就可以了：
//封装图片数组，这里是给的数据，你也可以换成url
NSMutableArray dataArr = [NSMutableArray new];
for (int i = 0; i < 5; i ++) {
Photo photo = [Photo new];
photo.image = [UIImage imageNamed:@”123”];
[dataArr addObject:photo];
{
Photo photo = [Photo new];
photo.image = [UIImage imageNamed:@”234”];
[dataArr addObject:photo];
}
}
//直接给数组，然后弹出来就可以了
BorewViewController borew = [BorewViewController new];
borew.photoArr = [dataArr mutableCopy];
borew.index = 2;
[self presentViewController:borew animated:YES completion:nil];
代码当然谁都可以看懂了，我这里就说下几个重要的地方就可以了：
一：BorewViewController中我没有一次性全部加在所有的图像，我是做了优化的，滚动到哪里就加载左右的就可以了；
二： if(!photo.image)
{
//异步获取图片信息
dispatch_async(dispatch_queue_create(0, 0), ^{
NSData *data = [NSData dataWithContentsOfURL:photo.url];
photo.image = [UIImage imageWithData:data];
});
}
scView.photo = photo;
这里我是把url获取到数据再给值，可以在显示所有图片时直接获取。
三：BorewScrollView中的缩放手势是直接用的滚动视图的，可以避免很多的问题，还是不错的；
四：[imageView setMultipleTouchEnabled:YES];这样才可以相应双击；
五：BorewScrollView中image的显示我是处理了的，

微信好像也是类似的处理吧（不知道）；
六：// 让UIImageView在UIScrollView缩放后居中显示 这个是直接设置的时候调用的

  • (void)scrollViewDidZoom:(UIScrollView )scrollView
    {
    CGFloat offsetX = (scrollView.bounds.size.width > scrollView.contentSize.width)?
    (scrollView.bounds.size.width - scrollView.contentSize.width)     0.5 : 0.0;
    CGFloat offsetY = (scrollView.bounds.size.height > scrollView.contentSize.height)?
    (scrollView.bounds.size.height - scrollView.contentSize.height) 0.5 : 0.0;
    imageView.center = CGPointMake(scrollView.contentSize.width     0.5 + offsetX,
    scrollView.contentSize.height * 0.5 + offsetY);
    }这里我是没看懂的，没加效果也是一样的，不过可以用它来做点击的位置居中显示的效果；
